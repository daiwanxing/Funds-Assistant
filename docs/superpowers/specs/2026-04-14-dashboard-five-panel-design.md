# Dashboard 五宫格重构设计

日期：2026-04-14

## 目标

将首页 `dashboard-page__detail` 区域从当前的详情切换区，重构为固定的五宫格驾驶舱布局。

本次调整只改右侧详情区域，不改左侧固定边栏的信息职责。

## 范围

保留：

- 顶部全球指数走马灯
- 左侧固定边栏
  - 搜索
  - 自选列表
  - 用户栏

重构：

- 右侧 `dashboard-page__detail` 容器内部布局

不在本次范围：

- 新闻真实数据接入
- 移动端或自适应窄屏方案
- 左侧边栏的信息结构重做

## 关键约束

- 首页默认展示左侧当前选中的基金数据
- `dashboard-page__detail` 最小宽度为 `1000px`
- 当视口不足以容纳右侧五宫格时，不做响应式压缩
- 容器允许横向滚动，优先保证驾驶舱布局完整性
- 右侧五宫格按桌面大屏固定比例设计

## 总体布局

页面保持双栏结构：

- 左侧：固定边栏
- 右侧：五宫格详情区

右侧五宫格分为两行：

### 第一行

- 左侧大卡：`业绩走势`
- 右侧中卡：`新闻板块`

### 第二行

- 左：`持仓信息`
- 中：`重仓股`
- 右：`行业分布`

## 卡片职责定义

### 1. 业绩走势

承担首页第一视觉与核心信息表达。

内容：

- 基金名称
- 基金代码
- 当前净值
- 当前所选周期的涨跌表现
- 周期下拉框
- 收益走势主图

布局：

- 左侧为大数字信息面板
- 右侧为图表主体
- 周期切换位于右上角，使用下拉框而不是横向页签

设计原则：

- 视觉上最亮、最大、最具存在感
- 参考驾驶舱主屏风格，不做传统后台图表卡
- 图表依然要保持足够横向空间，不能因信息堆叠而显得局促

### 2. 新闻板块

本次仅做占位，不接入真实新闻。

内容：

- 标题
- `Coming soon` 或同等含义状态标签
- 2 到 3 行占位条

设计原则：

- 明确表达这是预留模块
- 不使用伪新闻标题，不做看起来已经上线的假数据
- 保持卡片完成度，但视觉权重低于主卡

### 3. 持仓信息

承担基金基础资料卡的角色。

内容：

- 基金类型
- 基金公司
- 基金经理
- 基金规模
- 交易状态
- 单位净值日期或相关截止信息

设计原则：

- 以信息摘要为主，不做复杂图形
- 比旧版“持仓摘要”更完整，但仍保持可扫读
- 与另外两张下排卡片在视觉权重上保持一致

### 4. 重仓股

承担列表型结构数据展示。

内容：

- 序号
- 标的名称
- 标的代码
- 仓位权重条
- 占比
- 今日涨跌幅

头部信息：

- 标题
- 覆盖率标签，例如 `93.0% 覆盖`
- 截止说明，例如“截至上季度末”

设计原则：

- 风格按参考图 1:1 复刻
- 使用表格式紧凑排布
- 每一行都要能直观看到权重和今日涨跌
- 卡片内部允许独立纵向滚动

数据策略：

- 持仓列表仍来自当前基金持仓明细数据
- 每只股票的“今日涨跌”通过现有股票行情能力补查
- 不依赖持仓原始表格中的空占位字段

### 5. 行业分布

承担行业配置的结构化展示。

内容：

- 行业名称
- 占比
- 小型占比条
- 顶部季度或覆盖率信息

设计原则：

- 不只是一根横条，要有列表感
- 和重仓股卡片保持同一体系
- 控制展示数量，避免列表过长导致密度失衡

## 交互规则

- 左侧切换基金后，右侧五宫格整体切换为对应基金的数据
- 布局保持稳定，只更新内容，不做大范围跳动
- 周期切换只影响 `业绩走势` 卡
- 单张卡获取失败时，只在卡内降级，不影响其他卡
- 新闻占位卡不提供点击行为
- 重仓股卡内部滚动，避免整页被长列表拉坏

## 状态设计

### 加载态

- 五张卡都保留骨架结构
- 不允许在加载时塌成空白区域
- 主卡加载态仍需保留大卡轮廓与图表区域占位

### 失败态

- 单卡失败单卡降级
- 使用简洁提示，不遮挡整页其他内容

### 空态

- 新闻卡为计划内占位，不算异常空态
- 重仓股与行业分布若无数据，应提供明确提示，但保持卡片结构存在

## 视觉方向

- 右侧五宫格整体要更接近“驾驶舱 / 数据大屏”，不是传统后台管理页
- 卡片间有明确边界，但边框不能过重
- 主卡视觉最强，其余四卡降一档
- 下排三张卡视觉权重一致，避免出现第二主卡
- 新闻卡完成度要高，但不能抢主视觉

## 组件边界建议

为避免首页组件继续膨胀，右侧五宫格应拆为独立容器和卡片组件。

建议结构：

- `DashboardDetailGrid`
- `DashboardPerformancePanel`
- `DashboardNewsPlaceholderPanel`
- `DashboardHoldingsInfoPanel`
- `DashboardTopHoldingsPanel`
- `DashboardIndustryPanel`

数据原则：

- 页面层继续持有当前选中基金状态
- 各卡片只接收自己需要的数据
- 复杂的数据拼装与补查逻辑放在 composable 或 API 层，不塞进页面模板

## 实施顺序建议

1. 先重构 `dashboard-page__detail` 的网格骨架
2. 将现有基金详情数据拆给 5 张卡
3. 完成主卡布局与周期下拉框替换
4. 完成重仓股卡的样式复刻
5. 补齐每只重仓股的今日涨跌
6. 接入新闻占位卡
7. 补齐加载态、失败态、空态

## 验收标准

- 左侧边栏完整保留
- 右侧详情区变为固定五宫格
- `业绩走势` 为最大主卡，左信息右图表
- 周期切换为下拉框
- `新闻板块` 为占位卡，含标签与占位条
- 下排三张卡从左到右为 `持仓信息`、`重仓股`、`行业分布`
- `重仓股` 卡显示每只股票当日涨跌幅
- 右侧详情区最小宽度为 `1000px`
- 宽度不足时使用容器横向滚动，不做响应式压缩
